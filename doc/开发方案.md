# 修仙仓库拍卖联机游戏开发方案（已落地版）

## 1. 系统架构设计

- 前端：React + Vite + Tailwind，状态分为房间准备、游戏中、整局结算三段。
- 后端：Node.js + Express + Socket.IO，代码位于 `server/index.js`，负责房间生命周期、游戏状态机、回合与轮次结算。
- 实时通信：WebSocket（Socket.IO 或 ws），所有轮次动作均由服务端权威结算并广播。
- 房间与游戏：房间 ID 长期存在；每次开局生成 gameId；整局结束后销毁 gameId，回到准备阶段。
- 随机一致性：按 gameId + roundId 生成种子；系统提示统一随机，道具/技能按 playerId 派生种子。

## 2. 核心数据结构

```ts
type Room = {
  roomId: string;
  ownerId: string;
  status: "准备中" | "游戏中";
  settings: RoomSettings;
  players: Player[];
  currentGameId?: string;
};

type Game = {
  gameId: string;
  roomId: string;
  status: "进行中" | "已结束";
  currentRoundNo: number;
  rounds: Round[];
};

type Round = {
  roundId: string;
  roundNo: number;
  bag: StorageBag;
  auction: Auction;
  hints: HintRecord[];
  settlement?: RoundSettlement;
};

type Auction = {
  bidRound: number;
  phase: "行动中" | "揭晓" | "结算";
  activePlayerIds: string[];
  bids: Record<string, number | null>;
  usedTools: Record<string, string | null>;
};

type Player = {
  playerId: string;
  nickname: string;
  roleId: string;
  ready: boolean;
  connected: boolean;
  managed: boolean;
  spiritStone: number;
  loadout: string[];
};
```

## 3. 状态流转

- 房间状态：创建 -> 等待加入 -> 准备中 -> 开局中(绑定 gameId) -> 整局结束 -> 回准备中。
- 游戏状态：未开始 -> 回合进行中 -> 回合结算 -> 下一回合 -> 整局结算 -> 销毁。
- 回合状态：生成储物袋 -> 多轮竞拍 -> 产生赢家或流拍 -> 回合结算展示。
- 轮次状态：提示 -> 行动（道具/竞价）-> 揭晓 -> 胜负判定 -> 下一轮或结束。

## 4. 关键算法

- 储物袋生成：先按境界概率抽样，再按区间长尾分布抽总格数；物品按从上到下优先填充。
- 价格与格子：价格使用重叠区间+长尾采样，保留少量圣品极值物品。
- 胜负判定：第1~4轮按倍率判定，第5轮起比大小，平局者进入加赛。
- 信息揭示：统一 HintEffect 模型，系统提示对全员一致，道具提示按玩家独立执行。
- 图鉴筛选：按类型、品质、形状、价格区间过滤，支持从仓库点击轮廓/品质联动。

## 5. 前端页面与组件规划

- 准备页：房主设置、角色选择、5孔位道具配置、准备/开始按钮。
- 游戏页：左侧玩家排名，中间仓库和操作区，右侧提示与聊天四分栏。
- 结算页：回合结算（盈亏、分摊/返利、入场券），整局结算（灵石排名）。
- 弹窗：图鉴弹窗、物品详情弹窗、道具说明、数字输入面板。

## 6. 后端 REST 接口

- POST `/api/rooms`：创建房间。
- POST `/api/rooms/:roomId/join`：加入房间。
- POST `/api/rooms/:roomId/ready`：准备/取消准备。
- PATCH `/api/rooms/:roomId/settings`：房主更新设置。
- POST `/api/rooms/:roomId/start`：开始新游戏并返回 gameId。
- GET `/api/rooms/:roomId/state`：拉取房间与游戏快照。

## 7. WebSocket 事件

- `room:sync`：全量房间同步。
- `game:started`：广播新 gameId。
- `round:started`、`round:settled`。
- `bid:submit`：提交竞价。
- `tool:use`：提交道具使用。
- `auction:revealed`：轮次揭晓。
- `chat:send`、`chat:new`。
- `player:disconnect`、`player:reconnect`。

## 8. 聊天重复显示排查

- 客户端：去重键 `messageId`，渲染层仅按 id 展示一次。
- 服务端：禁止同一连接重复 ack 导致重复广播；广播统一由服务端发一次。
- 重连补偿：拉取增量消息时携带 `lastMessageId`，服务端只回补缺失区间。

## 9. 部署与分享

- 前端：Vercel/Netlify，设置 `VITE_API_BASE_URL`、`VITE_WS_URL`。
- 后端：Render/Railway/Fly.io，暴露 HTTPS 与 WSS。
- 本地联机测试：
  - 后端本地启动后用 ngrok/cloudflared 暴露端口。
  - 前端 `.env` 指向公网 `https://` 与 `wss://` 地址。
  - 多人测试建议统一时区、清理浏览器缓存、检查 WebSocket 断线重连。

## 10. 当前项目运行方式

- 启动后端：`node server/index.js`
- 启动前端：`npm run dev`
- 默认地址：
  - 前端：`http://localhost:5173`
  - 后端：`http://localhost:3001`
- 前端连接地址配置：
  - 默认读取 `window.__WS_URL__`，未设置时回退 `http://localhost:3001`
  - 部署时可在 `index.html` 注入 `window.__WS_URL__ = "wss://你的后端域名"`
